mmj.lang

Class Formula

  • java.lang.Object
    • mmj.lang.Formula
  • Direct Known Subclasses:
    LogicFormula, VarHypFormula

    public class Formula
extends java.lang.Object
    Formula is, basically just an array of Sym, with a counter.

    Sub-Classes of Formula exist: VarHypFormula and LogicFormula but their function is basically just to simplify certain coding in VarHyp, LogHyp, etc. In theory, the distinction would be useful for creating default exprRPN's.

    Formula: contains Cnst in sym[0], followed by the formula's "expression", which consists of zero or more Cnst's and Var's in sym[1]...sym[cnt - 1].

    Formula Factoids:

    • A formula must have length >= 1.
    • sym[0], the first Sym in a Formula must be a Cnst.
    • sym[0] is the Type Code of the Formula, and by extension, the Stmt's Type Code.
    • sym[1] through sym[sym.length - 1] are referred to in mmj as the Formula's "Expression" -- which is simply the Formula minus its Type Code.
    • Every Stmt has a Formula, but a Formula can be created without a corresponding Stmt (which is the reason Formula was made into a class.)

      • getParseNodeHolderExpr

        public ParseNodeHolder[] getParseNodeHolderExpr(VarHyp[] varHypArray)
        Builds a "custom" version of an Expression in which an array of ParseNodeHolders is output for use in generating a ParseTree.

        The key fact about the output ParseNodeHolders is that a Cnst in the Formula Expression just goes into the holder's "mObj" -- it will not be part of the ParseTree and the ParseNode element is null.

        On the other hand, Variables in the Formula's Expression are converted into ParseNodes with Stmt = the VarHyp; the ParseNodeHolder's "mObj" element is set to the VarHyp Stmt reference -- and this ParseNode will be part of the ultimate Parse Tree. In effect, we're "parsing" VarHyps and creating their output ParseNodes at this time.)

        Parameters:
        varHypArray - Array of VarHyp for Formula.
        Returns:
        ParseNodeHolder array.
        Throws:
        java.lang.IllegalArgumentException - if unable to find a VarHyp for one of the Formula's Var's.

      • buildRuleFormatExpr

        public Cnst[] buildRuleFormatExpr(VarHyp[] varHypArray)
        Builds a "rule format" version of the Formula's Expression. Each Cnst in the Expression is output unchanged, while the Type Code (a Cnst) of each Var is output instead of the var -- this requires looking up the Var's VarHyp in the input varHypArray. For example, Expression "( ph -> ps )" is output as "( wff -> wff )".
        Parameters:
        varHypArray - Array of VarHyp for Formula.
        Returns:
        ruleFormatExpr in an array of Cnst.
        Throws:
        java.lang.IllegalArgumentException - if unable to find a VarHyp for one of the Formula's Var's.
